<h2><a href=https://www.codewars.com/kata/69f02fccf0cb40beed6059ac/train/csharp target="_blank">The Zero-Allocation Cache (ValueTask & Records)</a></h2><h3>7 kyu</h3><h3 id="the-educational-challenge">The Educational Challenge</h3><p>In modern high-performance C#, allocating memory on the Heap causes the Garbage Collector (GC) to work harder, slowing down your application.</p><p>When building a Cache, you want it to be <strong>fast</strong> and <strong>allocation-free</strong> for repeat requests.If a method returns a standard <code>Task&lt;T&gt;</code>, it creates an object on the Heap <em>every single time it is called</em>, even if the data is already in your cache! </p><p>To fix this, C# introduced <strong><code>ValueTask&lt;T&gt;</code></strong>, which is a <code>struct</code>. If you return a <code>ValueTask&lt;T&gt;</code> from a cache hit, it returns instantly on the Stack with <strong>zero heap allocations</strong>.</p><p>C# also introduced <strong><code>record</code></strong> types, which are perfect for caching because they represent immutable data with built-in value equality.</p><h3 id="your-task">Your Task</h3><p>You are building a caching layer for a database. </p><ol><li>Create a <code>record</code> named <code>UserProfile</code> that contains two properties: <code>Id</code> (int) and <code>Name</code> (string).</li><li>Complete the <code>CachedUserProvider</code> class.</li><li>It receives an <code>IDatabase</code> in its constructor.</li><li>Implement <code>GetUserAsync(int id)</code>. <ul><li>If the user is in your local dictionary cache, return it immediately (this will secretly use the fast-path of <code>ValueTask</code>).</li><li>If the user is <em>not</em> in the cache, <code>await</code> the <code>IDatabase.FetchUserNameAsync(id)</code> method, create a new <code>UserProfile</code>, store it in the cache, and return it.</li></ul></li></ol><h3 id="example">Example</h3><pre><code class="language-csharp"><span class="cm-comment">// First call: Hits the slow database.</span><span class="cm-variable">UserProfile</span> <span class="cm-variable">user1</span> <span class="cm-operator">=</span> <span class="cm-keyword">await</span> <span class="cm-variable">provider</span>.<span class="cm-variable">GetUserAsync</span>(<span class="cm-number">1</span>); <span class="cm-comment">// Second call: Returns INSTANTLY from the cache.
