<h2><a href=https://www.codewars.com/kata/69b9359e8af0beedadc87db9/train/javascript target="_blank">Witcher's Contract: The Bestiary</a></h2><h3>7 kyu</h3><h3 id="lore">Lore</h3><p>Geralt has taken on a new contract, but the peasants, as usual, didn't understand much. They cannot describe the creature itself, but they managed to tell him what it fears and hides from. You need to help Geralt check his Bestiary and accurately identify what he is hunting, so the witcher can properly prepare his oils and potions.</p><h3 id="your-task">Your Task</h3><p>Write a function <code>identifyMonster(weaknesses, bestiary)</code> that matches the monster's fears from the peasants' tales with the knowledge from the Bestiary. The function must return the name of the monster whose vulnerabilities include all the weaknesses listed by the peasants.</p><h3 id="input">Input</h3><p><code>weaknesses</code>: A list of weaknesses noticed by the peasants. For example: <code>["Igni", "Silver"]</code>.</p><p><code>bestiary</code>: A reference guide where the key is the monster's name and the value is an list of its known vulnerabilities.</p><h3 id="output">Output</h3><p><code>monster_name</code> (String): The name of the identified monster.</p><p>If no creature from the Bestiary fits the description, the function should return the string <code>"Unknown monster"</code>.</p><h3 id="example">Example</h3><pre><code class="language-javascript"><span class="cm-keyword">const</span> <span class="cm-def">bestiary</span> <span class="cm-operator">=</span> {  <span class="cm-string cm-property">"Griffin"</span>: [<span class="cm-string">"Grapeshot"</span>, <span class="cm-string">"Hybrid Oil"</span>, <span class="cm-string">"Aard"</span>],  <span class="cm-string cm-property">"Noonwraith"</span>: [<span class="cm-string">"Yrden"</span>, <span class="cm-string">"Moon Dust"</span>, <span class="cm-string">"Specter Oil"</span>],  <span class="cm-string cm-property">"Drowner"</span>: [<span class="cm-string">"Igni"</span>, <span class="cm-string">"Necrophage Oil"</span>]};<span class="cm-variable">identifyMonster</span>([<span class="cm-string">"Igni"</span>, <span class="cm-string">"Necrophage Oil"</span>], <span class="cm-variable">bestiary</span>); <span class="cm-comment">// Returns: "Drowner"</span><span class="cm-variable">identifyMonster</span>([<span class="cm-string">"Yrden"</span>, <span class="cm-string">"Moon Dust"</span>], <span class="cm-variable">bestiary</span>); <span class="cm-comment">// Returns: "Noonwraith" (since these vulnerabilities are in its list)</span><span class="cm-variable">identifyMonster</span>([<span class="cm-string">"Silver"</span>, <span class="cm-string">"Garlic"</span>], <span class="cm-variable">bestiary</span>); <span class="cm-comment">// Returns: "Unknown monster"</span></code></pre><h3 id="notes--constraints">Notes / Constraints</h3><ul><li>The <code>weaknesses</code> list will always contain at least one element.</li><li>A monster is considered a match if it has <strong>all</strong> the weaknesses from the provided <code>weaknesses</code> list (even if the Bestiary lists additional vulnerabilities for it).</li><li>Uniqueness: If a match exists in the bestiary, it is guaranteed to be unique (no two monsters will match the exact same provided list of weaknesses).</li></ul>
