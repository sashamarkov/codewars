<h2><a href=https://www.codewars.com/kata/691ca4c58c5f7e662d508d31/train/python target="_blank">Saddle Up!</a></h2><h3>7 kyu</h3><h3 id="task">Task</h3><p>Find all the <strong>saddle points</strong> of a non-empty matrix of integers. A saddle point is an element that is minimal in its row and maximal in its column. Return them in a list of <code>(row,column)</code> coordinates. The order of the saddle points in the list is irrevelant.</p><h3 id="example">Example</h3><p>Consider the following matrix:</p><pre><code>6 4 37 0 24 3 25 3 3</code></pre><ul><li>The row minimums are <code>3, 0, 2, 3</code> (at positions 1 and 2).<br></li><li>The column maximums are <code>7, 4, 3</code> (at positions 0 and 3).<br></li><li>Therefore the <code>3</code>'s in the 3rd column are saddle points, but the <code>3</code>'s in the 2nd column are not.<br></li><li>Return <code>[(0,2), (3,2)]]</code> or <code>[(3,2), (0,2)]</code>.</li></ul><h3 id="constraints">Constraints</h3><p>Number of rows <code>r</code> and number of columns <code>c</code> satisfy<code>1 ≤ r,c ≤ 500</code>. So you should think about efficiency (a little).</p><h3 id="note">Note</h3><p>Matrix saddle points are used in game theory to identify optimal strategies in two-person zero-sum games - see <a href="https://themba.institute/operations-research/saddle-points/" data-turbolinks="false" target="_blank">here</a>. There is a related saddle point concept for functions, which has even wider applications - see <a href="https://en.wikipedia.org/wiki/Saddle_point" data-turbolinks="false" target="_blank">here</a>.</p>
