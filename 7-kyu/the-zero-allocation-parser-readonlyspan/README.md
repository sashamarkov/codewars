<h2><a href=https://www.codewars.com/kata/69f19b47bea7232e6b7dee22/train/csharp target="_blank">The Zero-Allocation Parser (ReadOnlySpan)</a></h2><h3>7 kyu</h3><h3 id="the-educational-challenge">The Educational Challenge</h3><p>In traditional C#, string manipulation (like using <code>.Substring()</code>, <code>.Split()</code>, or creating new strings) is incredibly slow. Every time you slice a string, C# allocates a completely new string object on the <strong>Heap</strong>. If you are parsing a massive file, this causes thousands of Heap allocations, forcing the Garbage Collector (GC) to freeze your app to clean up the mess.</p><p><strong>The Architectural Solution:</strong>C# 7.2 introduced <strong><code>Span&lt;T&gt;</code></strong> and <strong><code>ReadOnlySpan&lt;T&gt;</code></strong>. A Span allows you to look at a "slice" of an existing string or array without allocating any new memory. It sits entirely on the <strong>Stack</strong>, completely bypassing the Garbage Collector.</p><h3 id="your-task">Your Task</h3><p>You have inherited a high-frequency financial parser. The method <code>ParseStockPrice</code> extracts a stock's price from a raw data string formatted like this: <code>SYM:AAPL|PRC:150.25|VOL:1000</code>.</p><p>The current developer is using <code>.IndexOf()</code> and <code>.Substring()</code>. Because this runs thousands of times per second, the Heap allocations are destroying the server's performance.</p><p><strong>Refactor the <code>ParseStockPrice</code> method to use <code>ReadOnlySpan&lt;char&gt;</code>.</strong> You must extract the price (the number between <code>PRC:</code> and <code>|VOL:</code>) entirely allocation-free. </p><h3 id="constraints">Constraints</h3><ul><li>You must change the parameter from <code>string</code> to <code>ReadOnlySpan&lt;char&gt;</code>.</li><li>You must use <code>double.Parse()</code> directly on the sliced Span to return the final number.</li><li>The test suite will barrage your method with massive data. If you convert the Span back to strings or arrays, the Garbage Collector will slow you down, and you will fail the performance timeout!</li></ul>
