<h2><a href=https://www.codewars.com/kata/5a4bba4e80eba8df2d00012d/train/javascript target="_blank">Total Time</a></h2><h3>6 kyu</h3><p>Write the function <code>totalTime()</code>, which accepts an array. The array contains an arbitrary number of strings, where each string represents a duration in time (hours, minutes, seconds). The function should add all the durations and return the total time, as a string, on the following format: <code>"(num) days, (num) hours, (num) minutes, (num) seconds"</code></p><h3 id="restrictions">Restrictions</h3><ul><li>Your function should only return days, hours, minutes and seconds if they are <code>&gt; 0</code></li><li>If the sum of all the values in the array are zero, your function should return <code>"0"</code> (a string)</li><li>The string values in the array will never represent negative numbers. I.e. the sum of all the values in the array will never be less than 0.</li><li>The array will always contain at least one string. There is no need to check for empty arrays.</li><li>You can assume that all strings in the array follow the following pattern: <code>"hh:mm:ss"</code>, where h - hours; m - minutes; s - seconds</li><li>Note that hours and minutes are optional, so a more precise (but harder to read) specification is: <code>[[hh:]mm:]ss</code></li><li>If a duration is less than 10 it will always be preceeded by a zero. I.e. expect to always get this format: <code>["01:01:01"]</code>. You will never get this format: <code>["1:1:1"]</code></li><li>Remember that each value in the string returned by your function should be followed by a comma (unless it is the last value).<ul><li>Wrong: <code>"1 day 3 hours 5 minutes"</code></li><li>Correct: <code>"1 day, 3 hours, 5 minutes"</code></li></ul></li></ul><h3 id="example-1">Example 1</h3><p>If hours and/or minutes are 0 they should be excluded from the answer.</p><pre><code class="language-javascript"><span class="cm-variable">totalTime</span>([<span class="cm-string">'60'</span>]); <span class="cm-comment">// --&gt; "1 minute"</span></code></pre><p>Note that the above answer is <code>"1 minute"</code> and not <code>"1 minute 0 seconds"</code></p><h3 id="example-2">Example 2</h3><p>The unit name should be pluralized by adding an "s" if its value is more than 1.</p><pre><code class="language-javascript"><span class="cm-variable">totalTime</span>([<span class="cm-string">'25:01:01'</span>]); <span class="cm-comment">// --&gt; "1 day, 1 hour, 1 minute, 1 second"</span><span class="cm-variable">totalTime</span>([<span class="cm-string">'50:02:02'</span>]); <span class="cm-comment">// --&gt; "2 days, 2 hours, 2 minutes, 2 seconds"</span></code></pre><p>Note that <code>"1 second"</code> does not end with an "s", while <code>"2 seconds"</code> does. Same goes for days, hours and minutes.</p><h3 id="some-final-examples">Some final examples:</h3><pre><code class="language-javascript"><span class="cm-variable">totalTime</span>([<span class="cm-string">'01:20'</span>,<span class="cm-string">'03:10'</span>]);    <span class="cm-comment">// --&gt; '4 minutes, 30 seconds'</span><span class="cm-variable">totalTime</span>([<span class="cm-string">'00:50'</span>,<span class="cm-string">'00:30'</span>]);    <span class="cm-comment">// --&gt; '1 minute, 20 seconds'</span><span class="cm-variable">totalTime</span>([<span class="cm-string">'01:20:00'</span>,<span class="cm-string">'40:00'</span>]); <span class="cm-comment">// --&gt; '2 hours'</span><span class="cm-variable">totalTime</span>([<span class="cm-string">'12:00:00'</span>,<span class="cm-string">'10:00:00'</span>,<span class="cm-string">'02:00:00'</span>]); <span class="cm-comment">// --&gt; '1 day'</span><span class="cm-variable">totalTime</span>([<span class="cm-string">'24:00:00'</span>,<span class="cm-string">'24:00:00'</span>,<span class="cm-string">'07'</span>]); <span class="cm-comment">// --&gt; '2 days, 7 seconds'</span></code></pre>
