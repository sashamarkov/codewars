<h2><a href=https://www.codewars.com/kata/62c1b0b1f8f15c00654c407a/train/go target="_blank">Merging N Channels</a></h2><h3>6 kyu</h3><h1 id="introduction">Introduction</h1><p>This kata is an advanced version of <a href="https://www.codewars.com/kata/62c1a121f8ce9500571d90fe" data-turbolinks="false" target="_blank">Merging Two Channels</a>. It's recommended to solve that one first, as it already solves half of this problem :)</p><p>Go has a powerful built-in data type: <strong>channel</strong>. In some cases you want process messages from different data sources (e.g. UDP packages, TCP messages, HTTP requests) and you want to consume all in the same way. Then it might make sense to merge multiple channels into a single one. This is what we do here.</p><h1 id="learning-goal">Learning Goal</h1><p>In this kata we learn to <strong>merge any amount of channels into a single one</strong> by utilizing <strong>concurrency</strong> patterns.</p><h1 id="task">Task</h1><p>Write a function <code>func Merge(c ...chan string) &lt;-chan string</code>, which takes <strong>any amount of channels</strong> and returns a new channel. All messages from the input channels must be forwarded to the new channel. Once all input channels are closed, also the returned channel must be closed.</p><p>The order of the forwarded messages doesn't matter, but you should consume from all incoming channels
