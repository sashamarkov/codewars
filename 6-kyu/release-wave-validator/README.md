<h2><a href=https://www.codewars.com/kata/69debe70a31b1b9851068261/train/python target="_blank">Release Wave Validator</a></h2><h3>6 kyu</h3><h1 id="release-wave-validator">Release Wave Validator</h1><p>Write <code>solve</code> that checks whether an ordered deployment plan respects all prerequisite rules.</p><p>A deployment plan is split into waves. All packages inside the same wave are released at the same time. A prerequisite pair <code>(a, b)</code> means package <code>a</code> may only be released after package <code>b</code>.</p><h2 id="function-contract">Function contract</h2><p><code>solve(waves: list[list[str]], prerequisites: list[tuple[str, str]]) -&gt; bool</code></p><p><code>waves</code> is an ordered list of deployment waves. Each inner list contains the package names scheduled for that wave.</p><p><code>prerequisites</code> is a list of <code>(package, required_package)</code> pairs.</p><p>Return <code>True</code> if the plan is valid. Otherwise return <code>False</code>.</p><p>A plan is valid only when all of these rules hold:</p><ul><li>Every package scheduled in <code>waves</code> appears exactly once across all waves.</li><li>Every package mentioned in <code>prerequisites</code>, on either side of a pair, appears in exactly one wave.</li><li>If <code>(a, b)</code> appears in <code>prerequisites</code>, then <code>b</code> must be in a strictly earlier wave than <code>a</code>.</li><li>Packages in the same wave cannot satisfy each other's prerequisites.</li><li>Duplicate prerequisite pairs may appear and do not change the answer.</li><li>Empty waves may appear and should be ignored.</li></ul><p>Constraints:</p><ul><li>The total number of package names across all waves can be as large as <code>50000</code>.</li><li>The number of prerequisite pairs can be as large as <code>100000</code>.</li></ul><h2 id="examples">Examples</h2><p><code>solve([['auth', 'db'], ['billing'], ['reports']], [('billing', 'auth'), ('billing', 'db'), ('reports', 'billing')]) -&gt; True</code></p><p><code>billing</code> depends on <code>auth</code> and <code>db</code>, and both are released earlier. <code>reports</code> depends on <code>billing</code>, which is also earlier.</p><p><code>solve([['auth', 'billing'], ['db']], [('billing', 'auth')]) -&gt; False</code></p><p><code>billing</code> and <code>auth</code> are released in the same wave, so the prerequisite is not satisfied.</p><p>Return whether the deployment plan is valid.</p>
