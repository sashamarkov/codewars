<h2><a href=https://www.codewars.com/kata/6a020680357f9fb3614e6502/train/python target="_blank">MIDI Files: Variable Length Quantity</a></h2><h3>6 kyu</h3><h1 id="introduction">Introduction</h1><p>In MIDI files, timestamps are stored as integers representing the number of "ticks" since the previous event in the file.</p><p>Because these timestamps are relative to one another, most values tend to be fairly small. In order to save space, MIDI files use a special integer encoding format called a <strong>Variable Length Quantity (VLQ)</strong>.</p><p>A MIDI VLQ stores an integer using one or more bytes:</p><ul><li>The lower 7 bits of each byte contain the actual data</li><li>The highest bit indicates whether another byte follows<ul><li><code>1</code> means another byte follows</li><li><code>0</code> means this is the final byte</li></ul></li></ul><p>For example:</p><pre><code>0x7F -&gt; 01111111</code></pre><p>Since the highest bit is 0, this is the final byte.</p><p>Removing the continuation bit leaves:</p><pre><code>1111111 = 127</code></pre><hr><p>Values larger than 127 require multiple bytes.</p><p>For example:</p><pre><code>[0x81, 0x00]</code></pre><p>in binary becomes:</p><pre><code>10000001 00000000</code></pre><p>Removing the continuation bits gives:</p><pre><code>0000001 0000000</code></pre><p>Which is:</p><pre><code>00000010000000 = 128</code></pre><p>So:</p><pre><code>[0x81, 0x00] represents 128</code></pre><hr><h1 id="task">Task</h1><p>Your task is to implement two functions:</p><pre><code>encode(n)decode(data)</code></pre><ul><li><code>encode</code> should convert an integer into a MIDI VLQ byte sequence</li><li><code>decode</code> should decode only the first complete MIDI VLQ contained in the input. Any additional bytes after the terminating VLQ byte should be ignored.</li></ul><p>All test values will satisfy:</p><p><code>0 &lt;= n &lt;= 0x0FFFFFFF</code></p><p>Which is the maximum value representable by a 4-byte MIDI VLQ.</p><hr><h1 id="examples">Examples</h1><p>Encode:</p><pre><code>encode(0)    == [0x00]encode(127)  == [0x7F]encode(128)  == [0x81, 0x00]encode(8192) == [0xC0, 0x00]</code></pre><p>Decode:</p><pre><code>decode([0x00])                   == 0decode([0x7F])                   == 127decode([0x81, 0x00])             == 128decode([0xFF, 0xFF, 0xFF, 0x7F]) == 268435455</code></pre>
