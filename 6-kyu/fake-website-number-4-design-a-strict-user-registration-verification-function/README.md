Function  accept 1 parameter <code>user</code>, it is an <code>object</code> that contains three items:<code>username password and email</code>. like this:</p><pre><code class="language-javascript">{<span class="cm-variable">username</span>:<span class="cm-string">"myjinxin"</span>,<span class="cm-variable">password</span>:<span class="cm-string">"12349876"</span>,<span class="cm-variable">email</span>:<span class="cm-string">"myjinxin@hotmail.com"</span>}</code></pre><h2 id="the-validation-rules">The Validation rules:</h2><ul><li><p>The username length range: 4-15 characters; These reserved names can not be registered: <code>administrator, admin, system, john</code>; It should ignore the case, <code>myjinxin</code> and <code>MYJINXIN</code> are the same username; </p><p>If the username is invalid, you need to return the following information:  </p><pre><code class="language-javascript"><span class="cm-string">"Error: username is too short."</span><span class="cm-string">"Error: username is too long"</span><span class="cm-string">"Error: username contains invalid characters."</span><span class="cm-string">"Error: username has been registered by other users."</span><span class="cm-string">"Error: username has been reserved by system."</span></code></pre><p>You should check the error points in the order of the above to make sure that the correct error message is returned.</p></li><li><p>The password length range: 6-15 characters; It can use letters and numbers; It can't be a weak password: 1) Increasing  characters. such as <code>123456 abcdef</code>; 2) Consisting of one or two repeating characters. such as <code>111111 aaaaaa 010101 ababab</code>; 3) Username and password can not be the same.</p><p>If the password is invalid, you need to return the following information:  </p><pre><code class="language-javascript"><span class="cm-string">"Error: password is too short."</span><span class="cm-string">"Error: password is too long"</span><span class="cm-string">"Error: password contains invalid characters."</span><span class="cm-string">"Error: password is a weak password."</span></code></pre><p>You should check the error points in the order of the above to make sure that the correct error message is returned.</p></li><li><p>The email has no limit with length; It can use letters, numbers and <code>_.@</code>; Its format must be <code>aaa@bbb.ccc</code>; system only accept email address from these website: <code>hotmail.com facebook.com gmail.com 163.com 126.com</code>, email address from other websites are invalid.</p><p>If the email is invalid, you need to return the following information:  </p><pre><code class="language-javascript"><span class="cm-string">"Error: email address contains invalid characters."</span><span class="cm-string">"Error: email address format is invalid."</span><span class="cm-string">"Error: email address is not allowed."</span></code></pre><p>You should check the error points in the order of the above to make sure that the correct error message is returned.</p></li><li><p>Finally, if there is no error in the registration information, you should return <code>"User registration success!"</code> and add the user information to the end of the array <code>userdata</code>.
