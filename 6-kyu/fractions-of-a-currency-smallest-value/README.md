<h2><a href=https://www.codewars.com/kata/55c4a2a2586d8706be0000d0/train/python target="_blank">Fractions of a currency' smallest value</a></h2><h3>6 kyu</h3><h3 id="background">Background</h3><p>In the world of investment banking, when the quantities bought and sold are so large that pricing is significant at less then the smallest available unit of 'real' currency.</p><p>This is why you'll often see things quoted with a pair of buy-sell prices for a single producty that are less than a cent.</p><p>Once of the niceties of trading this way is a format to display these extra small fractions.  By convention, some products will be quoted not to decimal places, but to pre-agreed fractions.</p><p>For example, if a product is quoted to the 16th of a cent you might see a price displayed like this: <code>109.28/7</code> which means $109.28 <sup>7</sup>/<sub>16</sub></p><p>Internally within these systems the price will still be in cents and fractions of a cent like so: <code>10928.4375</code>.</p><h3 id="kata">Kata</h3><p>In this kata you must write a class to convert between internal and display values.</p><p>When constructed the class is supplied with the fractional denominator to be used in the conversion, if none is supplied, use <code>16</code> as a default.</p><p><strong>Note:</strong> You can assume that all inputs are valid for conversion.  Further, all denominators will be a power of two so no need to worry about floating point approximations.</p>
