<h2><a href=https://www.codewars.com/kata/56d2eada54d686d034000516/train/javascript target="_blank">Angular date formatter</a></h2><h3>6 kyu</h3><p>Angular provides a filter to correctly display the dates.</p><p>For this kata, we will only take into consideration the date and the format ( and let the timezone aside )</p><p><a href="https://docs.angularjs.org/api/ng/filter/date" data-turbolinks="false" target="_blank">Here is the official documentation about the filter</a></p><p>We simplify the possible representations to the following :</p><ul><li>**yyyy** : 4 digit representation of a year,padded (1970-9999)</li><li>**yy**   : 2 digit representation of a year, padded (00-99)</li><li>**MM**   : Month in a year, padded (01-12)</li><li>**M**    : Month in a year (1-12)</li><li>**dd**   : Day in a month, padded (01-31)</li><li>**d**    : Day in a month (1-13)</li><li>**HH**   : Hour in a day, padded (00-23)</li><li>**H**    : Hour in a day (0-23)</li><li>**mm**   : Minute in hour, padded (00-59)</li><li>**m**    : Minute in hour (0-59)</li><li>**ss**   : Second in minute, padded (00-59)</li><li>**s**    : Second in minute (0-59)</li></ul><p>the function dateFilter should take two arguments :<code>date</code> and <code>format</code>.</p><p><code>date</code> can be the <code>Date</code> object, a number, or a String (convertible in a number)</p><p><code>format</code> is the format of the output. Here are some examples :</p><pre><code class="language-javascript"><span class="cm-string">'HH:mm'</span> <span class="cm-comment">// Can return something like '12:04'</span><span class="cm-string">'dd/MM/yyyy'</span> <span class="cm-comment">// Can return something like '01/02/1995'</span><span class="cm-string">'d/M/yy H%m'</span> <span class="cm-comment">// Can return something like '1/31/14 1%59'</span></code></pre><p>Where there is an odd number of letters, parse the longest first then the remaining if needed.For example, <code>yyy</code> shoud be parsed as <code>[yy]y</code>, <code>ddd</code> should be parsed as <code>[dd][d]</code>.</p><p>When there are more than 2 of MdHms, for example, <code>dddd</code> it should be parsed as <code>[dd][dd]</code>.</p>
