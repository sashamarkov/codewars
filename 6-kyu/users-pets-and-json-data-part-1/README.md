<h2><a href=https://www.codewars.com/kata/64775418ac16620042e2efce/train/sql target="_blank">Users' pets and JSON data: Part 1</a></h2><h3>6 kyu</h3><p>You have been provided with a PostgreSQL table named <code>users</code>. This table includes a <code>jsonb</code> column, <code>info</code>, which holds JSON data. Here is a simplified schema of the table:</p><ul><li><code>id</code>: primary key, integer.</li><li><code>info</code>: JSON column which includes:<ul><li><code>name</code>: a string (user's name).</li><li><code>pets</code>: an array of JSON objects (each object represents a pet and has a <code>name</code> field and <code>type</code> field).</li></ul></li></ul><p>Your task is to construct a SQL query that will return a list of users and their pet names, but only for pets whose name starts with the letter <code>M</code>. If a user has no such pets, the user should not appear in the result table.</p><p>The query should return the following columns:</p><ul><li><code>id</code> - the users id</li><li><code>user_name</code> - the user's name</li><li><code>pet_names</code> -  the names of the pets whose names start with 'M' as a comma-separated list in the order that they appeared in the array.</li></ul><p>Result should be ordered by id in asc order.</p><p>Good luck!</p><h3 id="desired-output">Desired Output</h3><p>The desired output should look like this:</p><pre><code>  id  | user_name       | pet_names        |------+-----------------|------------------|  15  | Kelley Ebert    | Moriah           |  16  | Hayley Schiller | Micheal, Magaret |...</code></pre>
