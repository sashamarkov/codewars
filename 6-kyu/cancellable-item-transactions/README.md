<h2><a href=https://www.codewars.com/kata/69bf57993a0061ef6d03c095/train/python target="_blank">Cancellable Item Transactions</a></h2><h3>6 kyu</h3><h2 id="overview">Overview</h2><p><strong>1. Transaction Log</strong></p><p>You're given a string of item transactions, for example:</p><ul><li>"2A3B4C1A2A"</li><li>"5X6X1Y2Z1Y8X"</li></ul><p>Each transaction consists of a number followed by a letter, indicating the quantity of an item.</p><p>For example, "4A" means 4 units of item A</p><p>The string should be processed from left to right</p><p><strong>2. Cancellation Rule</strong></p><p>Sometimes, a letter appears on its own. This means that the most recent transaction of that item is <em>removed</em>, if one exists.</p><p><em>If there is no recent transaction for that item, the operation has no effect</em></p><p><strong>3. Pricelist</strong></p><p>You're also given a pricelist (hashmap/dictionary), mapping each item to its unit price</p><h2 id="task">Task</h2><p>Your task is to calculate the total price using the transaction string and pricelist</p><h2 id="constraints">Constraints</h2><ul><li>Multiple consecutive cancellations may occur</li><li>Negative quantities will be tested</li><li>Quantities with multiple digits will be tested</li></ul><h2 id="example">Example</h2><pre><code class="language-text">price_dict = {"A":1,"B":3,"C":2}transaction = "1A2BA3AC4CA"Steps (processed from left to right):  - "1A" -&gt; 1*1 = 1  - "2B" -&gt; 2*3 = 6  -  "A" -&gt; removes the most recent transaction of A (1A)  - "3A" -&gt; 3*1 = 3  -  "C" -&gt; no previous transaction of item C (nothing to remove)  - "4C" -&gt; 4*2 = 8  - "A" -&gt; removes the most recent transaction of A (3A)    Total: 14</code></pre>
